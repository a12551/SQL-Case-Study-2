 SQL Case Study – Employee & Department Data Analysis

## Overview
This project is a comprehensive SQL case study designed to demonstrate practical database management and data analysis skills using relational databases.

The case study focuses on analyzing organizational data related to employees, departments, job roles, and office locations using SQL queries and database operations.

---

## Objectives
The primary objectives of this project were to:

- Understand relational database structure
- Practice SQL querying techniques
- Perform employee and salary analysis
- Analyze departmental workforce distribution
- Work with joins, subqueries, and aggregate functions
- Generate business-oriented insights from structured data

---

## Database Tables Used

The project consists of the following tables:

- Employee
- Department
- Job
- Location

These tables are connected using relational database concepts and foreign key relationships.

---

## SQL Concepts Covered

### 1. Simple Queries
- Retrieving employee, department, job, and location details
- Selecting specific columns
- Using aliases

### 2. WHERE Conditions
- Filtering records
- Pattern matching using `LIKE`
- Using `BETWEEN`, `IN`, and logical operators

### 3. ORDER BY Clause
- Sorting data in ascending and descending order
- Multi-column sorting

### 4. GROUP BY & HAVING
- Department-wise salary analysis
- Job-wise salary analysis
- Employee count analysis
- Monthly and yearly hiring trends

### 5. JOINS
- Employee and department mapping
- Department and location analysis
- Workforce distribution across locations

### 6. Conditional Statements
- Salary grade categorization
- Employee grouping based on salary ranges

### 7. Subqueries
- Maximum salary analysis
- Second highest salary retrieval
- Department-wise employee filtering
- Average salary comparison

This case study helped in understanding:

- Salary distribution across departments
- Hiring trends by month and year
- Employee allocation by department and city
- Workforce analysis for business decision-making
- Department performance based on employee count
